  1. Start the API (with local PostgreSQL)

  Your .env is already configured for local PostgreSQL. Start the database first:

  # Start PostgreSQL in Docker
  docker run -d --name ugoki-pg -e POSTGRES_PASSWORD=dev -p 5432:5432 postgres:16

  # Run migrations and start API
  cd /Users/Berzins/Desktop/AI/UGOKI/ugoki_1_0/apps/api
  uv run alembic upgrade head
  uv run uvicorn src.main:app --reload

  2. Point Mobile to Local API

  Change your mobile .env to use localhost:

  # In apps/mobile/.env
  EXPO_PUBLIC_API_URL=http://localhost:8000

  Then start the mobile app:

  cd /Users/Berzins/Desktop/AI/UGOKI/ugoki_1_0/apps/mobile
  bun run start

  ---
  Alternative: Use SQLite (Simpler)

  If you don't want to run Docker/PostgreSQL locally, switch to SQLite:

  # In apps/api/.env, change:
  DATABASE_URL=sqlite+aiosqlite:///./ugoki.db

  This is simpler for development since SQLite requires no setup.

  ---
  Quick Reference

  | Environment | API .env DATABASE_URL                                     | Mobile .env EXPO_PUBLIC_API_URL |
  |-------------|-----------------------------------------------------------|---------------------------------|
  | Local Dev   | sqlite+aiosqlite:///./ugoki.db                            | http://localhost:8000           |
  | Local + PG  | postgresql+asyncpg://postgres:dev@localhost:5432/postgres | http://localhost:8000           |
  | Production  | Fly.io secret                                             | https://ugoki-api.fly.dev       |

  To start developing locally:

  # Terminal 1 - Start API
  cd apps/api
  uv run uvicorn src.main:app --reload

  # Terminal 2 - Start Mobile
  cd apps/mobile
  bun run start
